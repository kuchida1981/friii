## Why

現在、バックエンドとフロントエンドの基盤が構築されましたが、コード品質を強制する自動的な仕組みが不足しています。AI Agent との協業において、無意識に規約から外れたコードやテストのないコードが混入するのを防ぎ、常にデプロイ可能な品質を維持するために、厳格な CI/CD ガードレールを導入します。

## What Changes

- **静的解析の厳格化**: `golangci-lint` (Go) および `ESLint` (React) の設定を強化し、プロジェクト全域でフォーマットとコード品質を自動チェックします。
- **テストカバレッジの強制**: バックエンドおよびフロントエンドのテストにカバレッジ計測を導入し、一定の閾値を下回る変更をブロックします。
- **ローカルガードレール (pre-commit)**: サーバーサイド (GitHub Actions) だけでなく、ローカルでのコミット前にも同様のチェックを強制し、開発サイクルの高速化と CI リソースの節約を図ります。
- **GitHub Actions 連携**: プルリクエスト時に自動でテスト・Lint・カバレッジチェックを実行し、ステータスを可視化します。

## Capabilities

### New Capabilities
- `ci-cd-automation`: GitHub Actions による自動テストと品質チェックのパイプライン。
- `local-quality-control`: `pre-commit` を用いたローカル環境での品質担保。

### Modified Capabilities
- `testing-infrastructure`: テスト実行時のカバレッジ計測要件の追加。
- `environment-setup`: 開発環境のセットアップ手順（pre-commit の導入等）の更新。

## Impact

- **開発ワークフロー**: `git commit` 時に自動チェックが走るようになります。
- **ビルド・テスト**: テスト実行時間がわずかに増加しますが、カバレッジレポートが生成されるようになります。
- **CI/CD**: GitHub Actions のワークフローファイルが追加/更新されます。
