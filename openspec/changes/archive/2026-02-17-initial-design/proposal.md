## 背景 (Why)

青色申告特別控除（最大75万円）の要件である「優良な電子帳簿」に対応した、フリーランスエンジニア向けの会計管理システムを構築します。既存の汎用的な会計ソフトよりも、エンジニアのワークフローに適したシンプルかつ堅牢なシステムを目指します。

## 変更内容 (What Changes)

- **不変性を備えた複式簿記エンジン**: 仕訳の修正を「赤黒処理」で行い、物理的な削除を禁止することで「優良な電子帳簿」の要件を満たします。
- **フリーランスエンジニア特化の入力**: 現金取引を排除し、振替伝票形式の入力を中心とした設計。
- **監査ログの自動生成**: 全てのデータ変更履歴を記録。
- **決算報告書の出力**: e-Taxでの申告を補助する損益計算書(PL)および貸借対照表(BS)の生成。
- **取引先管理**: 申告時の売上内訳入力を容易にするための取引先（クライアント）情報の保持。

## ケイパビリティ (Capabilities)

### 新規ケイパビリティ
- `accounting-core`: 複式簿記の基本ロジック、仕訳管理、年度末繰越機能。
- `excellent-book-compliance`: 「優良な電子帳簿」要件（訂正・削除履歴、相互関連性、検索性）の担保。
- `financial-reporting`: 損益計算書、貸借対照表、総勘定元帳の生成。
- `client-management`: 取引先情報の管理と、仕訳への取引先紐付け。

## 影響範囲 (Impact)

- **Backend**: Go + GraphQL (gqlgen) による新規実装。
- **Frontend**: React + TypeScript (Vite) による新規実装。
- **Infrastructure**: Docker Compose による開発・実行環境の提供。
- **Data Storage**: PostgreSQL による履歴保持を重視したスキーマ設計。
